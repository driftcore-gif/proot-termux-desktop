What you get

Everything in one installer

One command sets up your full Linux desktop — distro, GPU, display, audio and apps all handled automatically.

🔍

Auto GPU Detection

Detects Adreno, Mali, NVIDIA, Intel, and Rockchip automatically. Applies the correct VirGL settings per device.

⚡

VirGL Acceleration

Hardware-accelerated rendering via virglrenderer-android. Confirmed 150+ FPS on Adreno 610.

Installation

Get running in minutes

Clone the repo, run the installer, follow the on-screen menus. That's it.

📱 Requires Termux from F-Droid (not Play Store) and Termux:X11 app installed on your device.
Termux — install
# Step 1 — Enable repos and update $ pkg install x11-repo root-repo $ pkg update && pkg upgrade # Step 2 — Clone repo $ pkg install git -y $ git clone https://github.com/driftcore-gif/Debian-termux-desktop.git ~/desktop $ cd ~/desktop # Step 3 — Run installer (interactive dialog menu) $ bash install.sh # Step 4 — Launch anytime $ tx11start

What the installer does

1

Enable x11-repo + root-repo, update package lists

2

Install Termux deps: proot-distro, termux-x11-nightly, virglrenderer, pulseaudio

3

Install selected proot distro (Debian/Ubuntu/Arch/Fedora/Void)

4

Generate tx11start launcher in /usr/bin — available globally

5

Auto-login to proot and run DISTRO-setup.sh (DE + apps selection)

Help

Frequently asked questions

Common issues and how to fix them.

Why does tx11start show a black screen? ▼
Usually caused by xfwm4 compositor conflicting with VirGL. The installer disables it automatically, but if you reset XFCE settings run:

xfconf-query -c xfwm4 -p /general/use_compositing -s false

Also make sure Termux:X11 app is open before running tx11start.
glxgears shows very low FPS or fails ▼
VirGL may not be running. Check with pgrep virgl_test_server. If empty, restart tx11start. Also verify your GPU was detected correctly — the boot log shows GPU: adreno or similar. If wrong, manually override in tx11start by adding GPU="adreno" near the top.
Desktop crashes or exits immediately ▼
This is usually the Android phantom process killer (Android 12+). Fix it with ADB:

pkg install android-tools -y
adb shell "device_config put activity_manager max_phantom_processes 2147483647"

Enable Wireless Debugging in Developer Options first.
No audio inside proot ▼
tx11start tries three PulseAudio modules in order: module-sles-source → module-aaudio-source → basic. If all fail, check that PulseAudio is running in Termux with pulseaudio --check. Make sure PULSE_SERVER=tcp:127.0.0.1 is exported inside proot.
Games not found / package not available ▼
Some games (Xonotic, OpenArena) are x86_64 only and automatically skipped on ARM devices. Run dpkg --print-architecture to check your arch. On armhf, use SuperTuxKart, Neverball, Freedoom and ExtremeTuxRacer instead.
Wine or Steam won't install ▼
Wine requires i386 multiarch support: dpkg --add-architecture i386 && apt update. Steam is x86_64 only and won't work on ARM Android devices at all. Both are marked experimental and may have compatibility issues depending on your device.
Termux says PREFIX is hardcoded at /data/user/0/... ▼
Termux is not supported on secondary Android user profiles. Switch to the primary user (User 0) and reinstall Termux from F-Droid. The Play Store version is outdated and will not work correctly.
How do I update after setup? ▼
In Termux: cd ~/desktop && git pull
Inside proot: apt update && apt upgrade

Re-run bash install.sh only if you want to change distro or connection type.
